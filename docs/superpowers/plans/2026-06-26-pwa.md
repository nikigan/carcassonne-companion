# PWA Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Carcassonne Companion installable to the home screen and usable offline for solo play, via `vite-plugin-pwa`.

**Architecture:** Add `vite-plugin-pwa` in `generateSW`/Workbox mode with `registerType: 'prompt'`. Workbox precaches the built app shell (solo state already lives in `localStorage`, so solo play works fully offline). A small React component surfaces a localized "update available → Refresh" toast; nothing reloads on its own. Multiplayer (WebSocket to the Durable Object) is untouched and inherently online. Static SW/manifest/icon files are served by the existing Cloudflare `ASSETS` binding.

**Tech Stack:** Vite 6, React 19, TypeScript (strict), Tailwind CSS v4, `vite-plugin-pwa` (Workbox), `@cloudflare/vite-plugin`.

**Design spec:** `docs/superpowers/specs/2026-06-26-pwa-design.md`

## Global Constraints

- **Build gate:** `npm run build` (`tsc -b && vite build`) MUST stay green. Run it before every commit.
- **Localize everything user-facing:** every new UI string gets a key in the `Strings` interface in `src/i18n.ts` with BOTH `en` and `ru` values (TypeScript enforces completeness). Russian uses correct grammar.
- **Do not break solo or multiplayer.** No changes to `src/useGame.ts`, `src/useRoom.ts`, `src/game/*`, `src/server/*`, or scoring.
- **Served from the domain root** (`base: '/'`); reference assets with root-relative paths.
- **Commits:** plain messages, imperative mood. Do NOT mention any AI assistant or add Co-Authored-By trailers (matches existing repo history).
- **Progress tracking:** maintain `.claude/pwa/PROGRESS.md`, updating it after each task (repo convention).

**Generation method (browser `<canvas>`, one-time):** `sharp`/`resvg` cannot render color-emoji glyphs, so rasterize the emoji with a real emoji font via a browser canvas. This is a one-time generation of static artifacts — no project dependency is added. The committed PNGs are the deliverable.

- [ ] **Step 2: Render the four canvases and capture base64 PNGs**

Run this with `javascript_tool` against that tab. It returns a JSON object of `{ filename: base64 }` (data-URL prefix stripped). The maskable variant uses extra padding so the emoji sits inside the platform mask safe zone; `apple-touch-icon` is full-bleed (iOS rounds it).

```js
(() => {
  const BG = '#1f2937'
  function draw(size, { radius, padding }) {
    const c = document.createElement('canvas')
    c.width = c.height = size
    const ctx = c.getContext('2d')
    // rounded (or square) dark background
    ctx.fillStyle = BG
    if (radius > 0) {
      const r = radius
      ctx.beginPath()
      ctx.moveTo(r, 0); ctx.arcTo(size, 0, size, size, r)
      ctx.arcTo(size, size, 0, size, r); ctx.arcTo(0, size, 0, 0, r)
      ctx.arcTo(0, 0, size, 0, r); ctx.closePath(); ctx.fill()
    } else {
      ctx.fillRect(0, 0, size, size)
    }
    // centered castle emoji, sized to leave `padding` fraction on each side
    const glyph = size * (1 - padding * 2)
    ctx.font = `${glyph}px "Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif`
    ctx.textAlign = 'center'
    ctx.textBaseline = 'middle'
    // optical nudge: emoji baseline box sits slightly high
    ctx.fillText('🏰', size / 2, size * 0.54)
    return c.toDataURL('image/png').split(',')[1]
  }
  return JSON.stringify({
    'pwa-192x192.png': draw(192, { radius: 38, padding: 0.16 }),
    'pwa-512x512.png': draw(512, { radius: 102, padding: 0.16 }),
    'pwa-maskable-512x512.png': draw(512, { radius: 102, padding: 0.26 }),
    'apple-touch-icon-180x180.png': draw(180, { radius: 0, padding: 0.14 }),
  })
})()
```

- [ ] **Step 4: Create the `UpdatePrompt` component**

Create `src/components/UpdatePrompt.tsx`:

```tsx
import { useRegisterSW } from 'virtual:pwa-register/react'
import { useI18n } from '../i18n'

/**
 * Bottom toast shown when a new app version has been precached. We use
 * registerType 'prompt', so nothing reloads until the user taps Refresh.
 */
export function UpdatePrompt() {
  const { t } = useI18n()
  const {
    needRefresh: [needRefresh, setNeedRefresh],
    updateServiceWorker,
  } = useRegisterSW()

  if (!needRefresh) return null

  return (
    <div
      role="status"
      className="fixed inset-x-0 bottom-0 z-40 flex justify-center px-4 pb-[max(1rem,env(safe-area-inset-bottom))]"
    >
      <div className="flex w-full max-w-md items-center justify-between gap-3 rounded-xl border border-white/10 bg-gray-800 px-4 py-3 shadow-2xl">
        <span className="text-sm text-white/90">{t.updateAvailable}</span>
        <div className="flex items-center gap-2">
          <button
            onClick={() => updateServiceWorker(true)}
            className="rounded-lg bg-white/15 px-3 py-1.5 text-sm font-semibold text-white hover:bg-white/25"
          >
            {t.refresh}
          </button>
          <button
            onClick={() => setNeedRefresh(false)}
            className="rounded-lg px-2 py-1.5 text-sm text-white/50 hover:bg-white/10"
            aria-label={t.close}
          >
            ✕
          </button>
        </div>
      </div>
    </div>
  )
}
```

## Notes for the implementer

- **Plugin order is the main risk.** If the build doesn't emit `dist/client/sw.js`, Task 2 Step 5's troubleshooting note covers reordering `VitePWA` vs `cloudflare`. Everything else is standard.
- **No runtime caching.** There are no external GET APIs to cache; multiplayer is a WebSocket (service workers don't intercept WS upgrades). The `navigateFallbackDenylist: [/^\/api\//]` is defense-in-depth.
- **Why `prompt` not `autoUpdate`:** a score tracker used live should never self-reload mid-game; the user explicitly chose prompt-to-refresh.
